# Cloudflare deployment

This site runs inside a Cloudflare Worker, using custom domains for production, staging and deploy previews. Each version of the site is a dedicated worker with its own DNS record and configuration. (We don't use `workers.dev` or branch deployments.)

## Production and staging

Both production and staging environments deploy using Cloudflare's CI, and their environments and build settings must be configured there. Make sure the following **build** variables are set:

- `NODE_ENV`: `staging` | `production`
- `PUBLIC_ENV`: `staging` | `production`

In addition, each worker needs a specific deployment command, which can only be set in the Cloudflare dashboard: Under `Build` > `Build configuration`, set the `Deploy command` to `npx wrangler deploy --config wrangler.production.jsonc` (or `wrangler.staging.jsonc` for staging).

If a specific Wrangler config file isn't set, Cloudflare will try to configure the project from scratch (because the default `wrangler.jsonc` doesn't exist). This can cause deploys to fail and other unforseen problems.

## Preview deploys

Preview configs are generated by `scripts/generate-preview-wrangler.sh` during the GitHub Actions workflow. Each PR gets its own worker and custom domain route. The worker is deleted when the PR is closed.

## Environments

| Environment | Domain                                    | `NODE_ENV`   | `PUBLIC_ENV` | Branch    | Deployed by                                             |
| ----------- | ----------------------------------------- | ------------ | ------------ | --------- | ------------------------------------------------------- |
| Production  | `www.documentcloud.org`                   | `production` | `production` | `main`    | Workers Builds (`wrangler.production.jsonc`)            |
| Staging     | `staging.documentcloud.org`               | `staging`    | `staging`    | `main`    | Workers Builds (`wrangler.staging.jsonc`)               |
| Preview     | `preview-<PR#>.staging.documentcloud.org` | `staging`    | `staging`    | PR branch | GitHub Actions (`.github/workflows/preview-deploy.yml`) |

## Local development

To run a local dev server against a specific config:

```bash
npx wrangler dev --config wrangler.staging.jsonc
```

### Key files

- `wrangler.production.jsonc` — production Workers Builds config
- `wrangler.staging.jsonc` — staging Workers Builds config
- `wrangler.preview.jsonc` — generated at CI time, not checked in
- `scripts/generate-preview-wrangler.sh` — generates preview config from PR number
- `.github/workflows/preview-deploy.yml` — GitHub Actions workflow for preview lifecycle
