# Wrangler Configs

Cloudflare Workers configuration files for each deployment environment.

## Files

| File               | Worker Name                      | Domain                                    | Deployed by                                           |
| ------------------ | -------------------------------- | ----------------------------------------- | ----------------------------------------------------- |
| `production.jsonc` | `documentcloud-frontend`         | `www.documentcloud.org`                   | Workers Builds (from `main`)                          |
| `staging.jsonc`    | `documentcloud-frontend-staging` | `staging.documentcloud.org`               | Workers Builds (from `main`)                          |
| `preview.jsonc`    | `dc-preview-<PR#>`               | `preview-<PR#>.staging.documentcloud.org` | GitHub Actions (generated at CI time, not checked in) |

## Environment variables

Production and staging builds require different env vars to select the right config in `src/config/`:

| Variable     | Production   | Staging / Preview |
| ------------ | ------------ | ----------------- |
| `NODE_ENV`   | `production` | `staging`         |
| `PUBLIC_ENV` | `production` | `staging`         |

Set these in the Workers Builds dashboard for production and staging. For previews, they're set in `.github/workflows/preview-deploy.yml`.

## Preview deploys

Preview configs are generated by `scripts/generate-preview-wrangler.sh` during the GitHub Actions workflow. Each PR gets its own worker and custom domain route. The worker is deleted when the PR is closed. See `plans/preview-deploy.md` for full details.

## Local development

To run a local dev server against a specific config:

```bash
npx wrangler dev --config wrangler/staging.jsonc
```
